Da der einfache Zugriff einer Andorid App über JDBC, wie man es in Java in der Regel macht, nicht funktioniert muss man das Ganze über WebSevices in Java machen. Also braucht man ein seperates Java Programm, in welchem die WebServices über einen Apache Server laufen. Zugegriffen wird dann über die jeweilige URL, je nach dem was man denn gerade braucht (siehe WebServices Dokumentation). Auf der anderen Seite ist dann die App. Die App greift auf die WebServices über die sogenannte Volley Bibliothek zu. Hierbei hat man einfach eine Klasse, die das Management für die Verbindung mit den WebServices übernimmt und die Responses immer in die jeweiligen Objekte (User, Food ...) parst und zurückgibt.
